Sujet

Résumé

Les buts du TP sont :
  • la programmation de array buffer permettant de décrire la géométrie (positions, normales) d'un objet 3D,
  • la compréhension du rôle de normales dans le rendu,
  • la compréhension du choix de localisation du calcul du rendu entre le vertex buffer et le fragment shader
    • .

Question 1 : modélisation d'une pyramide

A partir de l'exemple du cube, vous codez la description d'une pyramide (à base carrée). Chaque face de la pyramide a les couleurs suivantes :
  • base : gris
  • face frontale : jaune
  • face droite : bleu
  • face gauche : vert
  • face arrière : rouge
Les normales sont quant à elles perpendiculaires aux faces. Dans le array buffer décrivant les normales, celles ci ne sont forcément normalisées, il est donc important de penser les normaliser dans le vertex buffer.

Question 2

Modifiez les normales de manière à ce que la normale du sommet soit verticale, et les normales des sommets de la base (appartenant aux faces latérales) soient alignées aux diagonales de la base. Les normales des sommets de la base restent inchangées.
Décrivez les changements que vous constatez sur le rendu de la pyramide.

Question 3

La géométrie reste inchangée. Le calcul de l'éclairement est transféré du vertex shader au fragment shader. Les vecteurs nécessaires au calcul de l'éclairement : mvPosition et transformedNormal calculés dans le vertex shader doivent être envoyés au fragment shader sous forme de varying.
Expliquez pourquoi le rendu est différent.
